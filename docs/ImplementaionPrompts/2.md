You are a senior .NET 8 backend architect.

Generate a production-ready **Job Processing API and architecture** for a project called **YtProducer**.

This system orchestrates long-running media generation jobs such as:

* music generation
* image generation
* music visualizer generation
* YouTube upload

Some jobs may run **for up to 2 hours**, so the architecture must support long-running background processing.

The system must support:

* job orchestration
* retries
* progress tracking
* worker heartbeats
* pipeline execution per track
* UI monitoring

Use **clean architecture concepts**, but keep implementation simple.

Use:

* .NET 8
* ASP.NET Core Minimal API
* EF Core
* PostgreSQL
* Background Worker Service
* Dependency Injection
* async/await everywhere

Do NOT add external queues like RabbitMQ or Kafka.
Use **database-based job queueing**.

Do NOT add MediatR.

Code must be clean and compilable.

PROJECT CONTEXT

The system generates YouTube videos using this pipeline:

GenerateMusic
→ GenerateImage
→ GenerateVisualizer
→ UploadYoutube

Each step is a **Job**.

Jobs belong to a **Track**.

Tracks belong to a **Playlist**.
