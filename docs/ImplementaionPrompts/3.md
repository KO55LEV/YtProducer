You are a senior .NET 8 backend architect.

You are working on a project called **YtProducer**.

The system orchestrates media generation pipelines for YouTube content:

* music generation
* image generation
* visualizer video generation
* YouTube upload

Some jobs may take **minutes or even up to 2 hours** to complete.

The database schema has not been deployed yet, so we can safely redesign it.

Your task is to implement a **robust PostgreSQL-based job orchestration engine** suitable for long-running background tasks.

Do NOT introduce external queues like RabbitMQ, Kafka, or Redis.

PostgreSQL must be used as the job queue.

---

SYSTEM GOAL

Build a **generic job system** that can orchestrate pipelines such as:

GenerateMusic
→ GenerateImage
→ GenerateVisualizer
→ UploadYouTube

Each step is represented as a **job**.

Jobs must support long execution times and survive worker crashes.

---

DATABASE DESIGN

Create a `jobs` table with the following properties.

The system must support:

• generic job targets
• worker leasing
• heartbeat for long jobs
• retry logic
• idempotency protection

Schema:

CREATE TABLE jobs (
id uuid PRIMARY KEY,

```
type varchar(32) NOT NULL,
status varchar(32) NOT NULL DEFAULT 'Pending',

target_type varchar(32),
target_id uuid,

job_group_id uuid,

sequence integer,

progress integer NOT NULL DEFAULT 0,

payload_json jsonb,
result_json jsonb,

retry_count integer NOT NULL DEFAULT 0,
max_retries integer NOT NULL DEFAULT 3,

worker_id varchar(256),

lease_expires_at timestamptz,
last_heartbeat timestamptz,

created_at timestamptz NOT NULL DEFAULT NOW(),
started_at timestamptz,
finished_at timestamptz,

error_code varchar(64),
error_message text,

idempotency_key varchar(128)
```

);

Indexes:

CREATE INDEX idx_jobs_poll
ON jobs(status, created_at);

CREATE INDEX idx_jobs_target
ON jobs(target_type, target_id);

CREATE INDEX idx_jobs_lease
ON jobs(lease_expires_at);

CREATE UNIQUE INDEX idx_jobs_idempotency
ON jobs(idempotency_key)
WHERE idempotency_key IS NOT NULL;

---

WORKER LEASING MODEL

Workers must acquire jobs using **lease-based locking**.

Worker polling query:

SELECT * FROM jobs
WHERE status = 'Pending'
ORDER BY created_at
LIMIT 1
FOR UPDATE SKIP LOCKED;

Worker must then:

1. set status = 'Running'
2. set worker_id
3. set lease_expires_at = now() + interval '10 minutes'
4. set started_at

Workers must periodically update:

last_heartbeat

If a worker crashes, the job becomes recoverable.

---

STUCK JOB RECOVERY

Jobs whose lease expired must be automatically recoverable.

Recovery rule:

status = 'Running'
AND lease_expires_at < NOW()

These jobs should be reset to:

status = 'Pending'
retry_count += 1

note some jobs could take 2-5 hours ! 

---

RETRY LOGIC

Jobs must retry automatically when:

retry_count < max_retries

On failure:

status = 'Retrying'

After exceeding retries:

status = 'Failed'

---

IDEMPOTENCY

Prevent duplicate job creation.

Use:

idempotency_key

Example:

SHA256(type + target_type + target_id + stable payload fields)

If a job with the same idempotency_key exists, do not create a duplicate.
