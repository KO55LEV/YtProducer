GOAL
=====================================================================

Implement an advanced "frame-based" gym music visualizer renderer.

Input:
- image_path (single JPG/PNG)
- audio_path (MP3 or WAV)

Output:
- MP4 video 1920x1080 @ 30fps (YouTube-friendly)
- Animated image looks alive (3D-ish drift + zoom + rotation + depth glow)
- Air/dust particles overlay
- Equalizer bars drawn manually per frame using FFT data
- Beat/energy drives punch, shake, glow
- temp job folder created per request, optional cleanup
- Generate analysis.json and frame PNG files in temp

We accept the performance cost; correctness and quality first.

NO network calls.

=====================================================================
PIPELINE (MANDATORY)
=====================================================================

1) Probe audio duration (ffprobe) -> duration_seconds

2) Analyze audio -> produce analysis.json:
   - fps = 30 (default)
   - frame_count = ceil(duration_seconds * fps)
   - For each frame i:
       time = i / fps
       compute FFT magnitude spectrum for a window centered at time
       compress spectrum into eq_bands bands (default 64)
       compute:
         bass = avg bands 0..(bands*0.15)
         mid  = avg bands (0.15..0.55)
         high = avg bands (0.55..1.0)
         energy = weighted avg
       compute beat:
         beat=true if energy is a local peak above moving average threshold (simple onset detection)
   - Save JSON with metadata + per-frame array.

analysis.json format:

{
  "version": 1,
  "duration_seconds": 183.24,
  "fps": 30,
  "frame_count": 5498,
  "sample_rate": 44100,
  "channels": 2,
  "eq_bands": 64,
  "frames": [
    {
      "i": 0,
      "t": 0.0,
      "bands": [0.12, 0.08, ...],
      "bass": 0.71,
      "mid": 0.33,
      "high": 0.14,
      "energy": 0.56,
      "beat": false
    }
  ]
}

All band values normalized to 0..1.

3) Frame rendering (generate PNG frames):
   - Use input image as texture.
   - Render each frame 1920x1080.
   - The image MUST be animated (no static video):
       A) Base Ken Burns zoom from 1.00 -> 1.08 across full duration
       B) Micro rotation drift: small sinusoidal rotation
       C) Parallax-ish fake 3D:
          - Create two layers from same image:
            - base sharp layer
            - blurred bloom layer behind (scaled up 1.1x)
          - Move layers at slightly different offsets over time (gives depth feel)
       D) Beat punch:
          - if beat=true, apply quick zoom pulse and slight camera shake for that frame (decay over next ~6 frames)
       E) Air/Dust:
          - render 100-300 particles (small soft circles) that drift slowly
          - particle intensity/opacity tied to high or energy
          - particles are deterministic given seed
       F) Vignette + slight film grain (subtle)
   - Equalizer bars:
       - Draw eq_bands vertical bars centered near bottom (or mid-lower).
       - Bar height = bands[k] with smoothing (attack fast, release slower).
       - Add glow: draw blurred bar behind or draw same bar with alpha multiple times.
       - Color palette gym-style: white core + cyan/magenta accents, but not too flashy.
       - Bars should respond strongly to bass.
   - Write frame_i to frames/frame_000001.png etc.

Use deterministic random:
- Create RNG seeded from request.seed (or audio file hash).
- Pre-generate particle initial positions/velocities.
- Keep them stable but drifting.

4) Encode video from PNG frames + audio into MP4 using ffmpeg:
   - Input: frames/frame_%06d.png
   - Audio input: original audio file
   - Output: mp4 in MEDIA_OUTPUT_DIR (or derived from request output path if you add it; but for now generate):
       ./outputs/{timestamp}-{8charGuid}.mp4

FFmpeg encoding requirements:
- 30 fps
- libx264
- yuv420p
- profile high
- -movflags +faststart
- video bitrate default 12M
- audio: AAC 320k, 48kHz (YouTube friendly)
- stop at shortest

Example args (build programmatically):
ffmpeg -y
  -framerate 30 -i frames/frame_%06d.png
  -i audio.mp3
  -c:v libx264 -profile:v high -pix_fmt yuv420p -r 30 -b:v 12M -movflags +faststart
  -c:a aac -b:a 320k -ar 48000
  -shortest
  output.mp4

Capture stderr to logs/ffmpeg_stderr.txt and also return stderr_tail (last 8000 chars).

=====================================================================
IMPLEMENTATION DETAILS
=====================================================================

- AudioAnalysisService:
  - Decode audio to float samples (mono mixdown is OK).
  - Implement FFT (Cooley–Tukey) or use an existing small FFT implementation in code (no heavy dependency needed).
  - Windowing: Hann window.
  - Frame hop = sample_rate / fps (e.g. 1470 samples at 44100/30).
  - FFT size = next power of two (e.g. 2048 or 4096).
  - Normalize magnitudes and apply log scaling for visual punch.

- Beat detection:
  - Maintain moving average energy over last ~1s
  - beat=true when energy > avg * 1.35 AND is local max vs neighbors

- FrameRenderService:
  - If using SkiaSharp:
     - Load image to SKBitmap
     - Render onto SKSurface each frame
     - Apply transforms via canvas (translate/scale/rotate)
     - Implement vignette via radial gradient overlay
     - Grain via noise texture or random alpha points
     - Particles as blurred circles
     - Equalizer bars as rectangles with glow

- DeterministicRandom:
  - Provide NextFloat(), NextInt(), etc.
  - Used for particles + small random drift variations
  - Must be stable for same seed

- Cleanup:
  - Always attempt cleanup in finally unless keep_temp=true
  - If encoding fails, return ok=false and include stderr_tail.
